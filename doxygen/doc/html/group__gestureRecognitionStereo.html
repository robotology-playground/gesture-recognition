A module that trains and recognizes gestures in real-time using 3DHOF and HOG descriptors and linear SVMs as classifiers.

Description

This module is able to train and recognize in real-time different gestures. 3DHOF and pyramid-HOG are used as descriptors.

Commands:

The commands sent as bottles to the module port /<modName>/rpc are described in the following:

INITIAL_POSITION
format: [pos]
action: since we don't have skeleton information in this case, we need to work using the difference between an initial status and the subsequent frames to retrieve information on where the gesture is happening. To this end, this command save the current frame as the initial position, which the successive frames will be compared with.

SET
format: [set param]
action: param can be value or threshold. For an initial background removal, it is possible to set a depth value and a threshold that allows setting to 0 all the pixels that are closer than value+threshold and that are farther than value-threshold.

SAVE
format: [save param]
action: this command allows saving in the linearClassifierModule all the features from this moment until STOP is commanded. The descriptors are directly saved in a folder called actionX, where X=param is an integer.

TRAIN
format: [train]
action: all the descriptors saved so far are fed into the linearClassifierModule and support vector machines are trained accordingly.

REC
format: [rec]
action: the module starts to recognize any action that is being performed.

WRITE
format: [write]
action: the module starts saving features on a .txt file located in the directory specified as outDir in the ResourceFinder.

STOP
format: [stop]
action: the module stops recognizing or saving.

Libraries

  • YARP libraries.
  • stereo-vision library.
  • OpenCV library.

Ports Created

  • / <modName>/rpc remote procedure call. It always replies something.
  • / <modName>/scores:i this is the port where the linearClassifierModule replies the gesture that has been recognized at every frame.
  • / <modName>/features:o this port outputs the features to be sent to the linearClassifierModule.
  • / <modName>/classifier:rpc this port sends rpc commands like train or save to the linearClassifierModule.
  • / <modName>/scores:o this port sends the gameManager the gestures that have been recognized.
  • / <modName>/ispeak this port sends sentences to iSpeak.
  • / <modName>/depth this port sends the segmented depth image.

Parameters

The following are the options that are usually contained in the configuration file, which is gestureRecognition.ini:

–name name

  • specify the module name, which is gestureRecognitionStereo by default.

–robot robot

  • specify the robot to be used, by default.

–HOFnbins HOFnbins

  • number of bins for the 3D histogram of flow. The final histogram will be HOFnbins*HOFnbins*HOFnbins. HOFnbins is equal to 5 by default.

–HOGnbins HOGnbins

  • number of bins for the HOG. It is 128 by default.

–HOGlevels HOGlevels

  • number of levels of the pyramid HOG. 3 by default.

–pool pool

  • pooling strategy. can be "concatenate" or "max". "concatenate is usually used, especially if a dictionary is not specified.

– outDir outDir

  • folder where the features will be saved, if the write command is sent to the rpc port.

– dictionary dictionary

  • if this is true, a dictionary will be used. In this case the dictionary is specified in another config file.

– threshold threshold

  • threshold on the value of the depth that is given.

– value value

  • depth value at which the person should be moving.

– config_disparity config_disparity

  • file where the disparity information is stored. It is icubEyes.ini by default.

If a dictionary is specified, there exist two other config files: dictionary_hog.ini and dictionary_flow.ini. Both of them have a GENERAL group that contains:

– dictionarySize dictionarySize

  • file where the disparity information is stored. It is icubEyes.ini by default.
