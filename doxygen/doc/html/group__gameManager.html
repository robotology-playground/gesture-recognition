The module that implements the rules of All Gestures You Can game.

It manages the interactions between gestureRecognition and actionPerformer module.

Description

This module manages the interactions between gestureRecognition and actionPerformer modules. It keeps track of the turns between the human and the robot, and communicates with both the recognition and the performer modules.

Commands:

The commands sent as bottles to the module port /<modName>/rpc are described in the following:

SAVE
format: [save param]
action: this command serves to save all the features in the linear classifier module, for the successive training procedure. param can be a number from 1 to the number of actions that you want to train. This command is used only when the gestureRecognitionStereo module is running, it is not valid for the gestureRecognition using kinect.

TRAINED
format: [trained]
action: this command is needed to let the linearClassifierModule train all the gestures saved so far. This command also is valid only for the gestureRecognitionStereo module, not for the one using kinect.

START
format: [start]
action: through this command the game starts. The module chooses the opponent that should start performing a gesture.

WIN
format: [win]
action: this command just communicates that the robot has won and terminates the game.

LOSE
format: [lose]
action: this command just communicates that the robot has lost and terminates the game.

OVER
format: [over]
action: this command forcely terminates the game.

DONE
format: [done]
action: this is to notify that the robot has just finished performing the sequence, so it's time for the human to replicate the gestures.

TURN
format: [turn]
action: this is to notify the robot that the human has just finished performing the sequence, so it's time for the robot to replicate the gestures.

Libraries

  • YARP libraries.

Ports Created

  • / <modName>/rpc remote procedure call. It always replies something.
  • / <modName>/performer:o it sends the sequence to perform to the actionPerformer module.
  • / <modName>/gestRec:o it sends command to the gestureRecognition module.
  • / <modName>/scores:i this port reads the recognized gestures.
  • / <modName>/outspeak this port sends sentences to the iSpeak module.

Parameters

The following are the options that are usually contained in the configuration file, which is config.ini:

–name name

  • specify the module name, which is gameManager by default.

–robot robot

  • the robot that should execute the gestures.

–stereo stereo

  • it is on when the gesture recognition module based on stereo is being used.
