A module that executes gestures specified in a config file as a set of arm joint positions.

Description

This module simply allows the robot to perform a set of gestures that can be specified through a set of arm joint positions as in the file app/conf/actions.ini.

Commands:

The commands sent as bottles to the module port /<modName>/rpc are described in the following:

ACTION
format: [action] <id>
action: the module receives the gesture IDs to be performed and execute them.

STOP
format: [stop]
action: the module stops executing the sequence.

Libraries

  • YARP libraries.

Ports Created

  • / <modName>/rpc remote procedure call. It always replies something.
  • / <modName>/out through this port the modules outputs when the execution of the sequence has finished.

Parameters

The following are the options that are usually contained in the configuration file, which is config.ini:

–name name

  • specify the module name, which is actionPerformer by default.

–robot robot

  • the robot that should execute the gestures.

–stereo stereo

  • it is on when the gesture recognition module based on stereo is being used.

Tested OS

Windows, Linux
